# 多摄像头接入与现场演示指南

## 📹 摄像头接入方案

### 1. 本地摄像头（USB/内置摄像头）

系统已支持自动检测和选择多个本地摄像头设备。

**使用方法：**
1. 点击摄像头列表右上角的"刷新"按钮，系统会自动检测所有可用的摄像头设备
2. 系统会自动为每个检测到的摄像头创建条目
3. 在摄像头列表中选择要使用的摄像头即可

**支持的功能：**
- ✅ 自动检测多个USB摄像头
- ✅ 支持前后置摄像头切换（移动设备）
- ✅ 实时视频流显示
- ✅ 自动车牌识别

### 2. 网络摄像头（IP Camera / RTSP流）

支持通过URL接入网络摄像头流。

**支持的流格式：**
- **MJPEG流**：`http://摄像头IP:端口/stream` 或 `http://摄像头IP:端口/video.mjpg`
- **HLS流**：`http://摄像头IP:端口/stream.m3u8`
- **RTSP流**：需要转换为HTTP流（见下方说明）

**添加步骤：**
1. 在摄像头管理面板点击"+"按钮
2. 选择"网络流"类型
3. 输入摄像头名称（如：停车场入口）
4. 输入流地址URL
5. 点击"添加"

**常见摄像头流地址格式：**
```
# 海康威视
http://用户名:密码@IP地址:端口/Streaming/channels/1/picture

# 大华
http://IP地址:端口/cgi-bin/snapshot.cgi

# 通用MJPEG
http://IP地址:端口/video.mjpg
http://IP地址:端口/stream
```

**RTSP转HTTP方案：**
由于浏览器不支持直接播放RTSP流，需要先转换为HTTP流：

**方案A：使用FFmpeg + Nginx**
```bash
# 安装FFmpeg和Nginx
# 使用FFmpeg将RTSP转为HLS
ffmpeg -i rtsp://摄像头地址 -c copy -f hls -hls_time 2 -hls_list_size 3 -hls_wrap 10 /var/www/html/stream.m3u8

# 然后使用 http://服务器地址/stream.m3u8 作为流地址
```

**方案B：使用Node Media Server**
```bash
npm install node-media-server
# 配置RTSP转HTTP-FLV
```

### 3. 视频文件（用于演示）

支持使用本地视频文件作为输入源，非常适合现场演示。

**使用方法：**
1. 在摄像头管理面板点击"+"按钮
2. 选择"视频文件"类型
3. 输入名称（如：演示视频1）
4. 点击"选择文件"，选择本地视频文件（支持mp4、webm等格式）
5. 点击"添加"

**演示建议：**
- 准备包含清晰车牌的测试视频
- 视频会自动循环播放
- 可以同时添加多个视频文件，切换测试不同场景

## 🎬 现场演示优化方案

### 方案1：使用视频文件演示（推荐）

**优点：**
- ✅ 稳定可靠，不受网络影响
- ✅ 可以准备多个测试场景
- ✅ 演示效果可控
- ✅ 无需真实摄像头设备

**步骤：**
1. 准备测试视频文件（包含清晰车牌）
2. 在系统中添加为"视频文件"类型的摄像头
3. 选择该摄像头，点击"开启识别"
4. 系统会自动识别视频中的车牌

### 方案2：使用真实摄像头

**准备事项：**
- USB摄像头或网络摄像头
- 确保摄像头画面中有车辆/车牌
- 测试网络连接（如使用IP摄像头）

**优化建议：**
- 调整扫描间隔：在设置中可以将扫描间隔调整为1-3秒，提高识别频率
- 调整置信度阈值：降低阈值可以识别更多车牌（但可能增加误识别）
- 使用运动检测：系统会自动跳过静止画面，提高效率

### 方案3：混合演示

可以同时配置：
- 1-2个视频文件（稳定演示）
- 1个真实摄像头（展示实时效果）

在摄像头列表中快速切换，展示不同场景。

## ⚙️ 性能优化建议

### 1. 扫描间隔设置

- **实时监控**：建议 1-3 秒
- **演示模式**：建议 2-5 秒
- **低性能设备**：建议 3-5 秒

### 2. 置信度阈值

- **高精度模式**：0.8-0.9（减少误识别）
- **标准模式**：0.7（默认，平衡准确率和识别率）
- **演示模式**：0.6（提高识别率，可能增加误识别）

### 3. 多摄像头管理

- 系统支持同时管理多个摄像头
- 但一次只能识别一个摄像头（避免资源浪费）
- 可以在摄像头列表中快速切换

### 4. 网络摄像头优化

- **MJPEG流**：系统会自动每100ms刷新，确保实时性
- **健康检查**：系统每5秒检查连接状态，自动标记离线摄像头
- **自动重连**：摄像头离线后，系统会尝试重新连接

## 🔧 故障排查

### 问题1：本地摄像头无法访问

**解决方案：**
1. 检查浏览器权限：确保已授权摄像头访问权限
2. 检查设备连接：确保摄像头已正确连接
3. 检查其他应用：关闭可能占用摄像头的其他应用
4. 刷新设备列表：点击"刷新"按钮重新检测

### 问题2：网络摄像头无法连接

**解决方案：**
1. 检查流地址：确认URL格式正确
2. 检查网络：确保可以访问摄像头IP
3. 检查认证：某些摄像头需要用户名密码
4. 测试流地址：在浏览器中直接打开URL测试
5. 检查CORS：某些摄像头可能不允许跨域访问

### 问题3：识别效果不佳

**优化建议：**
1. 调整摄像头角度：确保车牌清晰可见
2. 改善光照条件：避免过暗或过亮
3. 调整扫描间隔：给AI更多处理时间
4. 降低置信度阈值：识别更多车牌（但需注意误识别）

## 🚀 快速开始

1. **启动系统**
   ```powershell
   .\start-dev.ps1
   ```

2. **添加演示视频**
   - 进入"实时监控"页面
   - 点击摄像头列表的"+"按钮
   - 选择"视频文件"类型
   - 选择测试视频文件

3. **开始识别**
   - 选择添加的视频摄像头
   - 点击"开启识别"按钮
   - 观察识别效果

4. **查看结果**
   - 识别结果会自动显示在界面上
   - 可以在"记录管理"页面查看历史记录
   - 可以在"数据仪表盘"查看统计信息
