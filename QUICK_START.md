# 快速开始 - 多摄像头与演示模式

## 🎯 核心改进

### ✅ 已实现的功能

1. **多本地摄像头支持**
   - 自动检测所有USB/内置摄像头
   - 支持在多个摄像头间切换
   - 记住每个摄像头的设备ID

2. **网络摄像头支持**
   - 支持MJPEG流（自动刷新）
   - 支持HLS流
   - 健康检查和自动重连

3. **视频文件输入（演示专用）**
   - 支持本地视频文件作为输入源
   - 自动循环播放
   - 完美适合现场演示

4. **摄像头管理**
   - 后端API持久化存储
   - 前端界面管理
   - 状态监控

## 🚀 3分钟快速演示

### 步骤1：准备测试视频（推荐）

1. 准备一个包含清晰车牌的MP4视频文件
2. 建议视频时长：30秒-2分钟
3. 确保车牌清晰可见

### 步骤2：启动系统

```powershell
# Windows
.\start-dev.ps1

# 或手动启动
# 终端1: cd ai-service && python main.py
# 终端2: cd backend && npm run dev
# 终端3: cd frontend && npm run dev
```

### 步骤3：添加视频摄像头

1. 打开浏览器访问 `http://localhost:5173`
2. 进入"实时监控"页面
3. 在右侧摄像头列表点击"+"按钮
4. 选择"视频文件"类型
5. 输入名称（如：演示视频）
6. 选择你的测试视频文件
7. 点击"添加"

### 步骤4：开始识别

1. 在摄像头列表中选择刚添加的视频摄像头
2. 点击"开启识别"按钮
3. 观察识别效果

### 步骤5：查看结果

- 识别结果会实时显示在界面上
- 可以在"记录管理"查看历史
- 可以在"数据仪表盘"查看统计

## 📹 接入真实摄像头

### USB摄像头

1. 连接USB摄像头到电脑
2. 在摄像头列表点击"刷新"按钮
3. 系统会自动检测并添加
4. 选择摄像头，点击"开启识别"

### 网络摄像头（IP Camera）

1. 获取摄像头的流地址（通常是 `http://IP:端口/stream`）
2. 在摄像头列表点击"+"按钮
3. 选择"网络流"类型
4. 输入名称和流地址
5. 点击"添加"

**常见流地址格式：**
- MJPEG: `http://192.168.1.100:8080/video.mjpg`
- HLS: `http://192.168.1.100:8080/stream.m3u8`

## ⚙️ 性能调优

### 扫描间隔设置

- **快速识别**：1000-2000ms（1-2秒）
- **标准模式**：2000-3000ms（2-3秒，默认）
- **省电模式**：3000-5000ms（3-5秒）

**调整方法：**
1. 点击右上角设置图标
2. 调整"扫描间隔"滑块
3. 保存设置

### 置信度阈值

- **高精度**：0.8-0.9（减少误识别）
- **标准**：0.7（默认，推荐）
- **演示模式**：0.6（提高识别率）

## 🔍 常见问题

**Q: 视频文件无法播放？**
A: 确保视频格式为MP4或WebM，浏览器支持的格式。

**Q: 网络摄像头连接失败？**
A: 
1. 检查流地址是否正确
2. 在浏览器中直接打开URL测试
3. 检查网络连接
4. 某些摄像头需要认证信息

**Q: 识别效果不好？**
A:
1. 确保车牌清晰可见
2. 调整摄像头角度
3. 改善光照条件
4. 降低置信度阈值

**Q: 如何同时监控多个摄像头？**
A: 目前系统一次只能识别一个摄像头。可以在摄像头列表中快速切换，系统会记住每个摄像头的状态。
