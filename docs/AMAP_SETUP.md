# 高德地图 API 配置指南

本系统已集成高德地图 API，支持真实地理位置选择和显示。

## 1. 申请高德地图 API Key

1. 访问 [高德开放平台](https://console.amap.com/)
2. 注册/登录账号
3. 进入「应用管理」->「我的应用」
4. 点击「创建新应用」
5. 填写应用信息，选择应用类型为「Web端(JS API)」
6. 创建 Key，选择服务平台为「Web端(JS API)」
7. 复制生成的 API Key

## 2. 配置 API Key

### 前端配置

在 `frontend` 目录下创建 `.env` 文件（如果不存在），添加：

```env
VITE_AMAP_KEY=你的高德地图API_Key
VITE_AMAP_SECURITY_CODE=你的高德地图安全密钥
```

**重要说明**：
- `.env` 文件不会被提交到 Git，请确保在本地创建
- **安全密钥（Security Code）**：自2021年12月02日起，高德地图新申请的API Key必须配备安全密钥一起使用，否则会出现 `INVALID_USER_SCODE` 错误
- 安全密钥可以在高德开放平台控制台的「应用管理」->「我的应用」->「Key」详情页面找到
- 如果遇到 `INVALID_USER_SCODE` 错误，请检查是否已配置 `VITE_AMAP_SECURITY_CODE`

### 验证配置

启动前端应用后，如果配置正确，在添加摄像头时点击「选择位置」按钮，应该能看到高德地图界面。

## 3. 功能说明

### 位置选择功能

- **搜索地点**：在搜索框中输入地点名称，可以搜索并选择位置
- **地图点击**：直接在地图上点击选择位置
- **标记拖拽**：选择位置后，可以拖拽标记调整精确位置
- **自动获取地址**：选择位置后，系统会自动获取该位置的详细地址

### 数据存储

系统会同时保存：
- **地址文本**：完整的地理位置描述（如：北京市朝阳区xxx路xxx号）
- **经纬度**：精确的坐标信息（latitude, longitude）

### 位置显示

在以下位置会显示真实地理位置：
- 摄像头列表中的位置信息
- 识别记录中的位置信息
- 车牌详情中的位置信息

## 4. 使用限制

高德地图免费版有一定的调用限制：
- 每日调用量限制（根据账号等级）
- 建议在生产环境中使用企业版以获得更高的调用配额

## 5. 故障排查

### 地图无法加载

1. 检查 `.env` 文件中的 `VITE_AMAP_KEY` 是否正确配置
2. 检查 API Key 是否已启用「Web端(JS API)」服务
3. 检查浏览器控制台是否有错误信息
4. 确认网络连接正常，可以访问高德地图服务

### 搜索功能报错 INVALID_USER_SCODE

1. **检查是否配置了安全密钥**：在 `.env` 文件中添加 `VITE_AMAP_SECURITY_CODE=你的安全密钥`
2. 安全密钥可以在高德开放平台控制台的「应用管理」->「我的应用」->「Key」详情页面找到
3. 配置安全密钥后，需要重启前端开发服务器
4. 如果仍然报错，请检查：
   - API Key 是否正确
   - 安全密钥是否与 API Key 匹配
   - 是否在加载地图脚本之前设置了安全密钥配置

### 位置选择后无法保存

1. 检查数据库表结构是否正确（应包含 `latitude` 和 `longitude` 字段）
2. 检查后端日志是否有错误信息

## 6. 安全建议

- **不要**将 API Key 提交到公开的代码仓库
- 在生产环境中，建议使用域名白名单限制 API Key 的使用范围
- 定期检查 API Key 的使用情况，防止异常调用
